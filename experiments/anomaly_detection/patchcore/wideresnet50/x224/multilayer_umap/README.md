# WRN50-2 Multilayer PatchCore UMAP Follow-up (`x224`)

This branch is curated as a reproducibility and results-review notebook built from the saved WideResNet50-2 multilayer PatchCore UMAP artifacts. By default the notebook does not retrain. It reloads the saved sweep tables, per-variant score CSVs, the selected variant checkpoint, and the saved embedding arrays for the selected UMAP run.

Notebook execution modes:
- `RETRAIN = False`: artifact-review mode. Load saved checkpoints, CSVs, and plots directly.
- `RETRAIN = True`: full rerun mode. The notebook launches `scripts/run_patchcore_wrn50_x224_umap.py` with the same runner, train config, data config, output dir, and worker argument pattern used by the Modal app.
- `REGENERATE_UMAP = False`: display the saved UMAP image directly.
- `REGENERATE_UMAP = True`: recompute the UMAP figure from the saved embedding `.npy` arrays and overwrite the selected variant image.

Key files:
- notebook: `experiments/anomaly_detection/patchcore/wideresnet50/x224/multilayer_umap/notebook.ipynb`
- train config: `experiments/anomaly_detection/patchcore/wideresnet50/x224/multilayer_umap/train_config.toml`
- artifact root: `experiments/anomaly_detection/patchcore/wideresnet50/x224/multilayer_umap/artifacts/patchcore-wideresnet50-multilayer-umap`

Artifact layout:
- `results/`: sweep tables, selected-variant review outputs, and the extracted legacy notebook images
- `plots/`: branch-level figures regenerated by the review notebook
- `topk_mb50k_r005_x224/checkpoints/best_model.pt`: selected saved checkpoint
- `topk_mb50k_r005_x224/results/evaluation/`: saved score CSVs and threshold sweep
- `topk_mb50k_r005_x224/results/umap/`: saved embedding, label, and score arrays for the UMAP follow-up
- `topk_mb50k_r005_x224/plots/`: selected variant plots, including the checked-in `umap_by_split.png`
