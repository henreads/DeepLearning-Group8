# PatchCore ViT-B/16 Fine-Tuned (`x224`)

This experiment fine-tunes the ViT-B/16 backbone on WM-811K normal wafers via a self-supervised rotation-prediction pretext task, then builds a standard PatchCore memory bank on the domain-adapted features.

## Method

**Stage 1 — Self-Supervised Fine-Tuning.**
The last 4 transformer blocks (blocks 8–11) and the final LayerNorm of a pretrained ViT-B/16 are updated using rotation prediction: each normal wafer is randomly rotated by 0°/90°/180°/270° and the model is trained to classify the applied rotation. No defect labels are used. Fine-tuning runs for 5 epochs with AdamW (LR = 3×10⁻⁵) and mixed-precision (fp16).

**Stage 2 — PatchCore Memory Bank.**
Block-9 patch tokens are extracted from the fine-tuned backbone, projected to 128 dimensions, and L2-normalised. A coreset-sampled memory bank of 600,000 tokens is built from the 40k training normals.

**Stage 3 — Threshold Calibration and Evaluation.**
The anomaly threshold is set by a sigma sweep on tune-normal scores, targeting 2% FPR with no defect leakage. Image-level scores are the mean of the top-10% patch distances to the memory bank.

## Notes

- The backbone pretraining initialisation is `vit_base_patch16_224.augreg_in21k_ft_in1k` (ImageNet-21k then ImageNet-1k).
- Blocks 0–7 remain frozen throughout; only blocks 8–11 and the LayerNorm are updated.
- The rotation accuracy converges from 83.5% (epoch 1) to 99.9% (epoch 5), confirming the model learns wafer-specific structure.
- This experiment contrasts with `one_layer_defect_tuning/` (frozen ImageNet backbone, defect-score threshold tuning) to isolate the contribution of domain-adapted features.
